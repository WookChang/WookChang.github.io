hexo+github博客搭建和优化

发表于 2018-01-23 | 分类于 项目 ， 博客 | 阅读次数

昨天整理里一些 Next 主题相关的信息，今天在记录一下前期建站的过程。

Hexo搭建

前期的搭建工作主要参考了一下几篇文章：

用 GitHub + Hexo 建立你的第一个博客

如何搭建一个独立博客——简明GithubPages与Hexo教程

当然更具体的细节可以参见 Hexo 的文档。仔细阅读文档后，了解了一些新的设定。比如在 Hexo 中的分类项和标签项存在着区别，分类项是有着上下级关系的树形结构；再比如可以设置模板项目，这样减少了每次新建文章修改 Font-matter 的工作量。

细节优化

关于mathjax

通过使用Mathjax可以解析网页上的数学公式，对于理工科的同学们是必备的。开启方法可以参见 Next 文档中关于第三方服务集成的介绍。Next 的版本号为5.0.1以上的可编辑 主题配置文件， 将mathjax下的enable设定为true即可。cdn用于指定 MathJax 的脚本地址，默认是 MathJax 官方提供的 CDN 地址。

但是之前存在过部分公式显示不出的问题，一度以为是输入公式格式存在问题，但是在其他 Markdwon 编辑器显示完好。后来才得知是 Markdwon 本身的特殊符号与Latex中的符号出现冲突。解决方法参考下面的文章：

Hexo下mathjax的转义问题

文中介绍了几种方法，我采用的是将 hexo 默认的渲染 markdown 的引擎换为 Pandoc。修改后公式显示就正常了。
