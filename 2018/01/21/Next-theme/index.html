Hexo主题之Next相关使用介绍

发表于 2018-01-21 | 分类于 总结 | 阅读次数

又是一年寒假来临，在阴雨的笼罩下，校园重启了空荡荡的模式，显得萧条冷落起来。今天是放假的第一天，做一点有意思的工作，把上个学期搭起来的blog整理整理。一条条的读完 Next 的使用文档，解决了之前公式显示中出现的问题，更换了第三方评论，并在其他一些细节个性化上做了小修小改。基本的操作方法在使用文档已经详细陈述了，本文主要对于主题设置的一些细节做一些补充。

Next 简介

最开始，我是在访问他人博客的时候发现地这份主题，被简洁的页面、优雅的排版所吸引。Next 这份主题有三种外观可以选择，支持多种语言，最关键的是集成了许多第三方服务，简化了建站的难度。回想之前曾在 VPS 上建站，从最底层开始搭起再应用 wordpress，对于我等小白来说耗时耗力，而如今就便捷的多了。

站点统计

站点统计使用的是百度统计。把相关的脚本id号粘贴到 主题配置文件 中的 baidu_analytics 字段即可。可以在后台清楚的看到站点的访问情况和流量分析等数据。

阅读次数统计

阅读次数统计（LeanCloud） 由 Doublemine 贡献。请查看 为NexT主题添加文章阅读量统计功能。发现的问题是，每次将本地文件 deploy 之后，阅读统计就会失效一段时间，过一段时间就好了。

评论系统

之前配置的是网易云跟帖，因为政策原因去年中旬已经取消服务。这次配置的是 liveRe 来必力 ，韩国出品。注册后获取 uid 粘贴到 主题配置文件 中的 livere_uid 字段即可。界面还是挺好看的，而且可以多平台账号登陆，管理页面也不错。存在的问题是有时后台中关于个人信息的页面打不开[捂脸]，还有就是载入时间有些长，经常看到logo图标在文末左晃右晃的。（— —/）

内容分享服务

据说现在gh-pages已经强制要求全站https，像百度分享就没有配置成功。最后采用的是AddThis提供的服务。在网站上注册账号，获得 AddThis id：More.. –> General –> ID 。在 主题配置文件 中，把#Share下的#add_this_id取消注释， 改为add_this_id: put_your_add_this_id_here 。图标也是挺好看的，国内的主流社交平台也都支持。

其他个性化改进

文末版权信息和圆形侧边栏头像的方法来自于基于Hexo+Next主题的个人博客搭建定制优化一文。

结语

现在对主题方面的设置比较满意了，说不定过一段时间会喜新厌旧，再做一些调整。但是，这些都不重要，重要的是减少在平台搭建上的消耗，多读书、多看论文、多一些内容产出。闻道有先后，术业有专攻。
